## üìö JWT Security Knowledge

### **What is JWT?**
JWT (JSON Web Token) is a compact, URL-safe token format used for securely transmitting information between parties as a JSON object. It is commonly used for **authentication** and **authorization** in web applications.

### **How JWT Works**
1. **User Login**: The user logs in with their credentials.
2. **Token Generation**: The server validates the credentials and generates a JWT.
3. **Token Storage**: The JWT is sent to the client and stored (usually in local storage or cookies).
4. **Token Verification**: For subsequent requests, the client sends the JWT in the `Authorization` header. The server verifies the token and grants access if valid.

### **Why JWT is Secure**
- **Signed Tokens**: JWTs are digitally signed using a secret (HMAC) or a public/private key pair (RSA/ECDSA).
- **Stateless**: JWTs are self-contained, meaning the server doesn‚Äôt need to store session data.
- **Expiration**: JWTs can have an expiration time (`exp` claim), after which they are no longer valid.

### **Best Practices for JWT Security**
- Use **HTTPS** to encrypt data in transit.
- Store JWTs securely (e.g., in HTTP-only cookies).
- Set a reasonable expiration time for tokens.
- Use strong secrets or keys for signing tokens.
- Implement token revocation for added security.
